# Live Parking And Mining

Estacionamento and Garimpeiro are not end-of-run notes. They are live
evolution mechanisms.

## Required Per Round

Every governed visual or harness-evolution round must produce one typed artifact
with:

- `parking_items`: blind spots, residues, failed assumptions, false-greens,
  blocked attempts or deferred risks;
- `mined_learnings`: reusable lessons that can become a rule, sensor, judge,
  contract, workflow change or skill update;
- `selection`: what survives, mutates, merges, parks or dies;
- `next_experiment`: the next smallest test that proves the adaptation.

## Failure Condition

If a round ends with only code changes and screenshots, the harness did not
learn. Return to Estacionamento/Garimpeiro before declaring the flow complete.
