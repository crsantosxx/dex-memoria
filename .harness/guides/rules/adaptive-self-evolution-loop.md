# Adaptive Self-Evolution Loop

Use this rule when a governed harness round needs to improve its own working
method while the product work continues.

The loop is:

1. Observe: capture the concrete failure, false green, delay or repeated loop.
2. Diverge: run the two-chamber meeting divergence roles before choosing a fix.
3. Converge: select one executable change, with owner, evidence and stop rule.
4. Patch: change the smallest useful rule, sensor, judge, guide, task or artifact.
5. Validate: run MCP sensors or judges against the active round target.
6. Select: keep what passed under pressure, park what did not, and name the next
   experiment.

## Required Artifact

Each adaptive loop needs an artifact with:

- `observed_failure`;
- `divergence_source`;
- `convergence_decision`;
- `changed_surface`;
- `validation_evidence`;
- `selection_result`;
- `removed_or_parked`;
- `next_experiment`.

The artifact must point to the active bad-loop, live parking/mining and
two-chamber meeting evidence.

## Survival Rule

A harness change survives only if it changes at least one future gate, sensor,
judge, rule, task, evidence requirement or stop rule. Pure credits,
descriptions or specialist mentions do not survive.
