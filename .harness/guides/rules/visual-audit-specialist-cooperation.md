# Visual Audit Specialist Cooperation

Use this rule during visual/layout audit rounds so specialists reduce risk
instead of adding ceremony.

## Rule

- Named specialist routing only applies when the environment declares a skills
  index/governance, such as `skills/INDEX.md` and
  `skills/00-governanca-mesa-de-skills.md`, or an equivalent local convention.
  Without that layer, ignore specialist names and use the phase, gate or screen.
- When that layer exists and this rule names a specialist, apply the lens or
  record why it did not apply. Do not downgrade named specialists to decorative
  comments or phase labels.
- If a visual audit source names an owner, preserve that owner. If it does not,
  use the phase, gate or screen as the central label.
- Fernanda do Fluxo routes phase, `next:` and `back_to:` and makes useful
  cooperators visible dynamically.
- Fernanda do Fluxo does not own visual content. She coordinates phase, owner,
  exit gate and transition; the domain specialist owns the visual finding.
- Kant coordinates the front line with Fernanda: keep patches small, demand
  current evidence, and block broad redesign without proof.
- Chato enters visual findings that can hide in green gates: icons, alignment,
  cuts, overlap, spacing gaps, omitted findings, card nesting and false
  positives.
- Focado owns the post-round meeting question: what should improve in the
  current workflow, flow gate, contract, guide, judge or sensor before the next
  round?
- Curioso asks whether the screen should optimize for the current operational
  action or for complete history/context. Park product choices when this is not
  clear.
- Atlas consolidates after multiple rounds by comparing sessions, tasks,
  sensors, judges, workflow changes and evidence folders.
- Akita Dev Raiz checks whether automation is proving reality or becoming
  dashboard theater.
- Garimpeiro separates reusable learning from local noise.
- Estacionamento holds useful residue that is not part of the current cut.
- Roteador Designer chooses the smallest design lens that fits the defect.
- Checklist Visual enumerates topbar, logo/theme icon, action icon, badge,
  alignment, clipping and responsive reachability checks.
- Dex Harness Project Setup joins when the improvement touches MCP proof,
  dashboard, workflow packaging, sensor registration or project sync.

## Material Credit

Do not merely say a specialist participated. Record a material contribution in
the session as a `review_finding`, `decision`, `flow_gate`, `judge_review` or
`sensor_run` that names the specialist, the phase and the evidence or result.

The dashboard should be read as evidence of material credits, not as a list of
every available specialist. If a useful specialist is missing from the
dashboard, add a session event that names the role and contribution instead of
inventing credit after the fact.

## Stop Condition

If the audit can only pass by trusting memory, old defaults or a dashboard
claim, return to evidence: current manifest, current screenshot, current task
status and current sensor/judge output.
